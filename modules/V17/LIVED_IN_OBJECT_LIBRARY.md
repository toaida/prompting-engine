# LIVED-IN OBJECT LIBRARY
### V17 — Proof-of-Life Objects by Space
### Status: ACTIVE — Runtime System

---

## PHILOSOPHY

**Core Problem:** AI-generated images show clean, staged spaces — no evidence of actual human habitation. Real spaces have the detritus of living: a half-drunk water bottle, a phone charger cable, a crumpled receipt.

**The Fix:** Add proof-of-life objects that prove this space is actually inhabited. Objects that tell a micro-story of human presence. NOT decoration — evidence.

---

## LIVED-IN OBJECTS BY SPACE

### 🏨 HOTEL ROOM

**Context:** Temporarily inhabited, but proof of current presence

**Primary Objects (must include 2-3):**
```
• Key card on table/desk (just arrived or checking out)
• Phone charger cable snaking from socket
• Water bottle half-empty on nightstand
• Room slippers worn and discarded by door
• Curtains pulled or open (not perfectly arranged)
```

**Secondary Objects (include 1-2):**
```
• Luggage open on floor or rack
• Clothes pulled from suitcase, not fully unpacked
• Coffee cup from room service
• Hotel pen and notepad
• Mini-bar fridge open or items removed
• Hotel Bible or menu on desk
```

**Avoid:**
```
❌ Perfectly made bed
❌ Suitcase fully unpacked and closed
❌ Nothing on desk/table
❌ All belongings hidden away
```

**Example Prompt Integration:**
```
"Hotel room, key card on desk, phone charging cable visible, 
room slippers kicked off by door, luggage open on floor"
```

## LIVED-IN OBJECT INTEGRATION RULES

### Rule 1: Objects Must Be In-Use State

```
BAD: "Water bottle on table"
GOOD: "Water bottle half-empty, condensation dried in patches"
BAD: "Phone on table"
GOOD: "Phone face-down, cable snaking to charger"
```

### Rule 2: Objects Tell Micro-Story

Each object should suggest:
- How long she's been here
- What she was doing
- What's about to happen
- Evidence of human imperfection

### Rule 3: Quantity Matters

Too few = staged emptiness
Too many = cluttered mess
Right amount = believable lived-in (3-5 objects visible)

### Rule 4: Match to Existence Reason

Objects should support the photo's existence reason:
- Waiting for friends → book, coffee, phone
- Just arrived → luggage, key card, looking around
- Resting → blanket, pillow, remote

### Rule 5: Imperfect Placement

```
BAD: "Items arranged neatly"
GOOD: "Items discarded or placed carelessly"
BAD: "Towel folded perfectly"
GOOD: "Towel spread, one corner folded over"
```
